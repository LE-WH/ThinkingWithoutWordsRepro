## Eval 架构（重要）

**不做 inline eval** — 训练和 eval 完全解耦：

```
训练阶段（DDP, 4 GPU）：
  Phase A → [保存 checkpoint 每 save_every 步] → merge
  Phase B → [保存 checkpoint 每 save_every 步] → merge

Eval 阶段（DDP 结束后，vLLM TP=4 自由使用 4 GPU）：
  eval_checkpoints.py → 对每个 checkpoint merge + vLLM eval → 记录到 W&B
```

**原因：** vLLM V1 会初始化 `torch.distributed(backend='nccl')`，即使 TP=1。与 DDP 现有 NCCL communicator 冲突，导致初始化挂起约 10 min 后超时。训练结束后 DDP 进程消失，vLLM TP=4 可以正常工作（无冲突）。

## Bug 修复记录

详见 `docs/20260516_smokereport.md`。关键修复：

1. **FA2 OOM** — `collate()` 返回 2D padding mask（`data_utils.py`）
2. **DDP "ready twice"** — two-pass backward 用 `no_sync()` 隔离（`train_phase_lora.py`）
3. **LR schedule 错误** — `total_steps × num_processes`（两个训练文件都已修复）
4. **vLLM inline eval 不可行** — 改为训练后批量 eval（架构变更）
5. **`eval_vllm_worker.py` 参数 bug** — `--begin-id` 等改为 optional，自动从 tokenizer 推导
