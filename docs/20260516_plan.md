# 训练计划

两个方案：Smoke Test（4× A100，~40 分钟）和 Full Training（4× B200，~9–20 小时）。

---

## 方案一：Smoke Test — 4× A100-80GB

### 目标

| 验证项 | 预期结果 |
|---|---|
| FlashAttention 2 生效 | 启动时打印 `attn_implementation: flash_attention_2` |
| Two-pass bottleneck 无报错 | Phase A loss 从 ~3.5 开始下降 |
| W&B 曲线正常 | `train/loss`、`train/loss_z`、`train/loss_y`、`train/grad_norm` 有数据 |
| In-training eval 触发 | `eval/acc`、`eval/mean_abstract_tokens` 出现在 W&B |
| Phase B 收敛 | loss 从 ~0.5 开始下降 |
| 全流程无崩溃 | run 跑完，final eval 输出 accuracy |

### 配置

```
N             = 5000        # 与原 smoke 相同
seq_len       = 8192
T             = 1           # 只跑 1 个 PI round（Phase A → Phase B）
LoRA rank     = 32
epochs        = 1
micro_batch   = 1
grad_accum    = 16          # effective batch = 4 GPU × 1 × 16 = 64
eval_every    = 20 steps    # Phase A ~78 steps → 触发 ~3 次 eval
eval_n        = 100         # 每次 eval 100 题，约 2-3 分钟
```

### 时间估算（4× A100 SM80，FA2 生效）

| 阶段 | 估算 |
|---|---|
| 环境检查 + 数据确认 | ~2 min |
| 基线 eval（可跳过：`SKIP_BASELINE=1`）| ~3 min |
| Phase A（78 steps × ~14s/step）| ~18 min |
| in-training eval × 3 次 | ~8 min |
| gen_traces Phase B（5k, vLLM TP=4）| ~1 min |
| Phase B（78 steps × ~1.5s/step）| ~2 min |
| final eval（vLLM TP=4, 500题）| ~1 min |
| **总计** | **~35–40 min** |

### 命令

**前提：**
```bash
cd /workspace/ThinkingWithoutWordsRepro
wandb login                      # 或设置 WANDB_API_KEY 环境变量
pip show wandb | grep Version    # 确认已安装
```

**一键运行（推荐）：**
```bash
WANDB_PROJECT=abstract-cot \
WANDB_RUN_NAME=smoke_4xA100 \
EVAL_DATA=$PWD/data/math500.jsonl \
EVAL_EVERY=20 \
EVAL_N=100 \
N=5000 T=1 EPOCHS=1 MAX_LEN=8192 \

```

> `run_smoke.sh` 会通过 `03_phase_a.sh` 和 `06_phase_b.sh` 将这些环境变量传入 `train_phase_lora.py`。

**或逐步手动运行（方便调试）：**

```bash

python scripts/download_data.py --n 5000 --skip-math500 \

bash scripts/03_phase_a.sh

# 5. Merge

bash scripts/05_gen_traces.sh

# 7. Phase B
BASE=runs/qwen3-4b-abs/pi1_phaseA_merged \
TRACES_FILE=runs/qwen3-4b-abs/pi1_phaseB_traces.jsonl \

```

### 验收标准

**控制台：**
```
attn_implementation: flash_attention_2          ← FA2 确认
[bottleneck] ep0 step 5/78 loss=3.xxxx (z=x.xxx y=x.xxx) gnorm=x.xxx ...
[eval] step=20 acc=xx.xx%  abs=xx.x  resp=xxx.x  total=xxx.x  n=100  t=xxxs
```

**W&B dashboard（`abstract-cot` project）：**
- `train/loss` 单调下降
- `train/loss_z` 和 `train/loss_y` 分离显示（Phase A 专属）
- `train/grad_norm` 在 0.3–2.0 之间波动（正常范围）
- `eval/acc` 随 step 变化（初始可能很低，smoke scale 下属正常）
- `eval/mean_abstract_tokens` 应在 5–30 之间

**Final eval 输出：**
```
ABSTRACT acc=xx.xx  mean_abs=xx.x  mean_resp=xxx.x  ...
```
数值不重要，pipeline 跑完不报错即通过。

## 方案二：Full Training — 4× B200-179GB

### 目标

尽量接近论文 arXiv:2604.22709 的 warm-up 结果（Table 1: Abstract-CoT Warm-up, 86.2%，168 tokens）。

### 配置

```
N             = 60000       # 预过滤 16k token 的完整 CoT
seq_len       = 16384
T             = 3           # 3 PI rounds
Full FT       = True        # 全参数微调（非 LoRA）— 使用 train_phase.py
epochs        = 1           # per phase
micro_batch   = 2           # per GPU；ZeRO-3 分片后每卡约 25-30 GB
grad_accum    = 16          # effective batch = 4 × 2 × 16 = 128
eval_every    = 100 steps
eval_n        = 200
```

**注意：Full FT 需要 `train_phase.py`（非 `train_phase_lora.py`）。** wandb + eval 的改动目前只在 `train_phase_lora.py` 里，需要移植过去（结构完全相同，约 20 分钟工作量）。移植前可先用 LoRA 版跑以验证数据和流程。

### 时间估算

| | SDPA（当前代码）| FA2 two-pass（Phase A 重构后）|
|---|---|---|
| Phase A × 3 rounds | ~17 hr | ~7 hr |
| Phase B × 3 rounds | ~1.2 hr | ~1.2 hr |
| gen_traces Phase A（60k, 2 rounds）| ~0.7 hr | ~0.7 hr |
| gen_traces Phase B（60k, 3 rounds）| ~0.2 hr | ~0.2 hr |
| eval + merge overhead | ~0.5 hr | ~0.5 hr |
| **总计** | **~20 hr** | **~10 hr** |

**推荐先跑 SDPA 版（无需改代码），如果时间可接受再做 FA2 重构。**

### 前置步骤（在 B200 机器上）

```bash
# 1. clone & 环境
git clone <repo>
cd ThinkingWithoutWordsRepro
bash scripts/setup.sh          # 自动装 torch 2.9.1 + vLLM 0.14.0 + flash-attn（SM90 B200 支持）

# 2. W&B 登录
wandb login                    # 或 export WANDB_API_KEY=<key>

# 3. 下载预过滤数据（16k token filter，~266k 条）
python scripts/download_data.py \
  --n 60000 \
  --max-total-tokens 16384 \
  --dolci-out data/dolci_60k_16k.jsonl \
  --buffer-size 20000

# 4. 下载 MATH-500（如果不存在）
python scripts/download_data.py --skip-dolci \
  --math500-out data/math500.jsonl

# 5. 从 HF 下载 base warm-up 模型（已 merge 的 pi3_phaseB）
#    或者从头开始训练（扩展 Qwen3-4B base model）
bash scripts/download_from_hf.sh  # 下载 runs_hf/final 作为参考
bash scripts/01_extend_model.sh   # 创建 runs/qwen3-4b-abs/base
```

### 命令

```bash
WANDB_PROJECT=abstract-cot \
WANDB_RUN_NAME=full_60k_16k_T3 \
EVAL_DATA=$PWD/data/math500.jsonl \
EVAL_EVERY=100 \
EVAL_N=200 \
N=60000 \
T=3 \
EPOCHS=1 \
MAX_LEN=16384 \
MICRO_BATCH=2 \
GRAD_ACCUM=16 \
DATA_FILE=$PWD/data/dolci_60k_16k.jsonl \
bash scripts/run_smoke.sh
```

> `run_smoke.sh` 的 `T=3` 会自动生成 on-policy teacher traces 并进行多轮 PI。

### W&B 关键 metrics

| Metric | 健康区间 | 异常信号 |
|---|---|---|
| `train/loss`（Phase A round 1 开始）| ~3.0–3.5 | >4 说明 Z̃ 初始化有问题 |
| `train/loss`（Phase A round 2 开始）| ~0.3–0.5 | 如果和 round 1 一样高，on-policy traces 没有起作用 |
| `train/loss_z` vs `train/loss_y` | 前者应高于后者 | 两者相同说明 pass 分离没生效 |
| `train/grad_norm` | 0.1–3.0 | 持续 >10 说明需要降 LR |
| `eval/mean_abstract_tokens` | 10–50 | 固定为 13 = trace 退化（同 LoRA smoke 的问题）|
| `eval/acc`（Phase B round 3 后）| 目标 >75% | 论文 86.2% |

### 与 LoRA Smoke 结果的对比

| | LoRA Smoke（已有）| Full Training 目标 |
|---|---|---|
| 数据量 | 5k | 60k（12×）|
| 参数方式 | LoRA 17% | Full FT 100% |
| seq_len | 8192（76% 截断）| 16384（56% 完整 CoT）|
| eval 准确率 | 72.0% | 预期 78–82% |
| mean total tokens | 432 | 预期 200–300 |

### 已知风险

1. **`train_phase.py` 没有 wandb/eval**：目前只改了 `train_phase_lora.py`。Full FT 上 wandb 需要移植，否则只有控制台输出。
2. **gen_traces 时间**：60k samples × Phase A teacher（12k token prefix）≈ 45 min per round with vLLM TP=4。2 rounds = ~90 min，是整个 run 里第二大开销。
3. **seq=16k 的显存**：4× B200 179 GB，ZeRO-3 Full FT 每卡约 30 GB，显存够用，但 Phase A 的 SDPA 2D mask 在 seq=16k 时每样本约 512 MB。micro_batch=2 时每卡约 1 GB mask，可以接受。
4. **FA2 two-pass 未实装在 `train_phase.py`**：如果要用 FA2 加速 Full FT，需要先把 two-pass 逻辑移植到 `train_phase.py`（约 1 小时工作量）。
